# Пропсы (props) в React: Понятие, особенности и примеры

Props (сокращение от «properties» — свойства) представляют собой основной механизм передачи данных в React, обеспечивающий связь между компонентами и позволяющий создавать гибкие, переиспользуемые интерфейсы

### Что такое пропсы и их назначение

В архитектуре React данные обычно передаются явно от родительского компонента к дочернему через пропсы. Это позволяет родителю настраивать поведение и внешний вид своих «потомков», передавая им необходимые значения, такие как строки, числа, объекты или функции

### Ключевые особенности пропсов

- Односторонний поток данных: Информация всегда движется сверху вниз по дереву компонентов. Это делает поток данных предсказуемым и упрощает отладку: если в дочернем компоненте отображаются неверные данные, их источник всегда можно найти выше по цепочке
- Неизменяемость (Read-only): С точки зрения дочернего компонента пропсы являются «только для чтения». Компонент никогда не должен изменять свои собственные пропсы; вместо этого он должен реагировать на те значения, которые передает ему родитель
- Реактивность: Пропсы являются реактивными значениями. Если родительский компонент обновляет значение, передаваемое в пропсе, React автоматически инициирует повторный рендеринг дочернего компонента, чтобы синхронизировать интерфейс с новыми данными
- Поверхностное сравнение: При обновлении React сравнивает старые и новые пропсы, используя алгоритм shallow equality (поверхностное сравнение через Object.is). Это важно учитывать при передаче объектов или массивов: если родитель создает новый объект при каждом рендере, React сочтет пропс изменившимся, даже если его содержимое осталось прежним

### Управляемые и неуправляемые компоненты

Понятие пропсов тесно связано с концепцией управления компонентами:

- Управляемый компонент (controlled): Важная информация в нем (например, состояние открытия панели или значение поля ввода) определяется пропсами, которые приходят от родителя. Это дает родителю полный контроль над логикой
- Неуправляемый компонент (uncontrolled): Компонент сам хранит свое состояние внутри (через useState), а родитель не может напрямую влиять на него

### Примеры использования пропсов

1. Передача простых данных
   Родительский компонент передает строку или число, а дочерний компонент использует их для отображения

```tsx
// Дочерний компонент "читает" пропсы
function Heading({ level, text }) {
  // Пропсы доступны только для чтения [1, 2]
  return <h1 className={`size-${level}`}>{text}</h1>;
}

// Родитель настраивает "потомка"
function Page() {
  return <Heading level={1} text="Заголовок страницы" />; // [3, 4]
}
```

2. Передача функций (обработчиков событий)
   Этот способ позволяет дочернему компоненту взаимодействовать с родителем, например, чтобы обновить состояние в родителе.

```tsx
function ActionButton({ onAction, label }) {
  // Вызов функции, переданной через пропсы
  return <button onClick={onAction}>{label}</button>;
}

function App() {
  const handleTrigger = () => alert("Действие выполнено!");
  return <ActionButton label="Нажми меня" onAction={handleTrigger} />;
}
```

3. Использование пропса children
   Специальный пропс children позволяет передавать JSX-разметку внутрь компонента, реализуя паттерн композиции

```tsx
function Card({ children }) {
  // children — это всё, что вложено между тегами компонента [7]
  return <div className="card-container">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Заголовок внутри</h2>
      <p>Текст внутри карточки.</p>
    </Card>
  );
}
```

### Основные правила в коде:

- Инкапсуляция: Дочерний компонент не знает, откуда пришли данные, он просто отображает их
- Реактивность: Если родитель изменит значение, которое он передает (например, level), React автоматически перерисует дочерний компонент с новым значением
- Компоненты верхнего уровня: Чтобы избежать «прокидывания пропсов» (prop drilling) через множество уровней, рекомендуется использовать композицию или Context API

### Ограничения: Prop Drilling и альтернативы

Когда данные нужно передать очень глубоко по дереву через компоненты-посредники, которым эти данные не нужны, возникает проблема «прокидывания пропсов» (prop drilling)

Для решения этой проблемы в React существуют альтернативные подходы:

- Композиция компонентов и пропс children: Вместо передачи данных вниз можно передать сами дочерние компоненты как JSX, используя специальный встроенный пропс children
- Context API: Позволяет «телепортировать» данные глубоко вниз к любому компоненту в дереве, минуя явную передачу через пропсы на каждом уровне
