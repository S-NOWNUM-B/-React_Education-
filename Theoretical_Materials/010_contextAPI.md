# Context API в React: Глубокая передача данных

Context API — это встроенный механизм React, который позволяет передавать данные через дерево компонентов без явного прокидывания пропсов на каждом уровне. Контекст дает возможность сделать определенную информацию доступной для любого компонента в поддереве, независимо от глубины его вложенности.

### Проблема, которую решает Context API

Обычно данные в React передаются от родителя к потомку через пропсы. Если одни и те же данные, например текущая тема или информация о пользователе, нужны компоненту глубоко в дереве, их приходится передавать через множество промежуточных компонентов.

Такой подход называется prop drilling. Он делает код более громоздким и усложняет поддержку. Context API позволяет передавать данные напрямую тем компонентам, которым они действительно нужны.

### Как работает Context API

Процесс использования контекста в приложении обычно состоит из трех шагов:

- **Создание контекста**: С помощью функции createContext создается объект контекста. Обычно его выносят в отдельный файл. В createContext можно передать значение по умолчанию, которое будет использовано, если компонент не найдет провайдер выше по дереву
- **Предоставление значения через Provider**: Родительский компонент оборачивает часть дерева в провайдер и передает значение через свойство value. Это значение становится доступным всем вложенным компонентам
- **Чтение значения через useContext**: Дочерний компонент вызывает useContext и получает текущее значение нужного контекста

### Когда Context API особенно полезен

Контекст подходит для данных, которые можно считать общими для части приложения:

- **Темизация**: Передача настроек светлой или темной темы по всему интерфейсу
- **Текущий пользователь**: Хранение информации об авторизованном пользователе
- **Роутинг**: Многие библиотеки маршрутизации используют контекст для хранения текущего пути и связанных данных
- **Общее состояние**: Данные, которые должны читаться и обновляться в разных частях дерева компонентов

### Масштабирование: Context API + useReducer

Для более сложных приложений Context API часто используют вместе с useReducer. Такой подход помогает отделить хранение состояния от логики его обновления.

Обычно схема выглядит так:

- Один контекст передает текущее состояние
- Второй контекст передает функцию dispatch

Это избавляет от необходимости прокидывать обработчики через пропсы и позволяет глубоко вложенным компонентам инициировать обновления состояния напрямую.

### Важные ограничения и рекомендации

- **Не использовать без необходимости**: Если данные нужно передать всего на 2-3 уровня вниз, обычные пропсы часто остаются более простым и понятным решением
- **Рассматривать композицию**: Иногда вместо контекста удобнее передать JSX через children и тем самым сократить количество промежуточных компонентов
- **Учитывать перерисовки**: При изменении значения контекста React перерисовывает все компоненты, которые читают этот контекст через useContext

### Пример использования Context API

Ниже базовый пример, где данные пользователя становятся доступны вложенным компонентам без ручной передачи через каждый уровень:

```tsx
import { createContext, useContext } from "react";

const UserContext = createContext(null);

function App() {
  const user = { name: "Анна" };

  return (
    <UserContext.Provider value={user}>
      <Page />
    </UserContext.Provider>
  );
}

function Page() {
  return <Profile />;
}

function Profile() {
  const user = useContext(UserContext);

  return <h1>Привет, {user.name}!</h1>;
}
```

### Заключение

Context API является важным инструментом для построения масштабируемых React-приложений. Он помогает избавиться от лишнего prop drilling и упрощает доступ к общим данным в глубоко вложенных компонентах. При этом использовать контекст стоит осознанно, когда данные действительно нужны многим частям дерева компонентов.
