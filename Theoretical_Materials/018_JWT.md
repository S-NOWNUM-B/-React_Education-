# Работа JWT (JSON Web Token) в React-приложениях

`JWT` (JSON Web Token) - это стандарт передачи подписанных данных между клиентом и сервером. В React-приложениях JWT чаще всего используется для token-based авторизации: сервер выдает токен после входа, а клиент отправляет его в защищенные запросы.

## Как работает JWT (workflow)

Типичный сценарий выглядит так:

1. Пользователь отправляет логин и пароль на сервер.
2. Сервер проверяет данные и возвращает токен.
3. Клиент сохраняет токен (например, в `localStorage` или cookie).
4. Клиент добавляет токен в заголовок `Authorization` для защищенных запросов.
5. Сервер проверяет подпись и срок действия токена, после чего возвращает данные.

Пример заголовка:

```http
Authorization: Bearer <token>
```

## Где JWT используется в React

- **Глобальное состояние**: хранение токена и статуса авторизации через Redux Toolkit или Context API.
- **Private Routes**: редирект на страницу входа, если пользователь не авторизован.
- **API-клиент**: автоматическая подстановка токена в запросы через Axios interceptors.

## Пример входа и сохранения токена

```tsx
async function login(email: string, password: string) {
  const response = await fetch("/api/auth/login", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ email, password }),
  });

  if (!response.ok) {
    throw new Error("Ошибка авторизации");
  }

  const data = await response.json();
  localStorage.setItem("token", data.token);
}
```

## Пример Axios interceptor

```tsx
import axios from "axios";

export const api = axios.create({
  baseURL: "/api",
});

api.interceptors.request.use((config) => {
  const token = localStorage.getItem("token");

  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }

  return config;
});
```

## Важные практические моменты

- **Срок действия токена**: access-токен обычно живет недолго.
- **Refresh token**: используется для продления сессии без повторного логина.
- **Безопасность**: важно учитывать XSS/CSRF-риски и выбранный способ хранения токена.

## JWT в дорожной карте разработчика

- **Junior**: чаще работает с открытыми API и базовыми формами.
- **Middle/Senior**: проектирует защищенную авторизацию, работает с access/refresh токенами и ошибками истечения срока действия.

## Инструменты отладки

Для проверки авторизации до интеграции во фронтенд обычно используют Postman или Insomnia:

1. Отправляют запрос на логин.
2. Копируют полученный токен.
3. Проверяют доступ к защищенному endpoint с заголовком `Authorization: Bearer <token>`.

## Заключение

JWT - базовый механизм авторизации в современных SPA. В React он тесно связан с глобальным состоянием, защищенными маршрутами и настройкой API-клиента.
