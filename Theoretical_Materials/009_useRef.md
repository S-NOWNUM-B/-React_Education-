# useRef в React: Ссылки на значения и манипуляция DOM

useRef — это встроенный хук React, который позволяет хранить информацию, не участвующую в отрисовке интерфейса. В отличие от состояния (state), изменение ссылки не вызывает повторный рендеринг компонента

### Основные характеристики

- Хранение значения: useRef возвращает объект с единственным свойством current, которое изначально равно переданному initialValue
- Стабильность: React гарантирует, что один и тот же объект ссылки будет сохраняться между всеми повторными рендерами одного и того же экземпляра компонента
- Локальность: Информация в ссылке является локальной для каждой копии компонента

### Для чего используется useRef

1. Доступ к DOM-узлам: Самый частый сценарий — манипуляция элементами напрямую (например, чтобы сфокусировать текстовое поле, прокрутить страницу к элементу или измерить его размеры)
2. Хранение технических данных: Ссылки идеально подходят для хранения информации, которая не влияет на визуальный результат: ID таймеров (setTimeout), интервалов или экземпляров внешних библиотек
3. «Инструмент побега» (Escape Hatch): Refs позволяют взаимодействовать с системами вне парадигмы React, например, с браузерными API

### Важные правила использования

- Запрет на чтение/запись во время рендеринга: Вы не должны обращаться к ref.current во время выполнения тела функции компонента. Читать или изменять ссылки следует только в обработчиках событий или внутри useEffect
- Иммутабельность vs Мутабельность: В то время как состояние компонента нельзя изменять напрямую, свойство ref.current является мутабельным — вы можете менять его значение вручную

### Примеры использования

1. Доступ к DOM-узлу (фокусировка)

```tsx
import { useRef } from "react";

function Form() {
  const inputRef = useRef(null); // Создаем ссылку

  function handleClick() {
    // В обработчике события обращаемся к DOM-узлу напрямую
    inputRef.current.focus();
  }

  return (
    <>
      <input ref={inputRef} /> {/* Привязываем ссылку к элементу */}
      <button onClick={handleClick}>Сфокусировать поле</button>
    </>
  );
}
```

2. Хранение ID интервала

```tsx
function Stopwatch() {
  const intervalRef = useRef(null);

  function handleStart() {
    // Сохраняем ID, не вызывая перерисовку экрана
    intervalRef.current = setInterval(() => {
      /* логика */
    }, 1000);
  }

  function handleStop() {
    clearInterval(intervalRef.current); // Читаем ID для остановки
  }
}
```

### Ограничения

По умолчанию собственные компоненты не выставляют ссылки на свои внутренние DOM-узлы. Если вам нужно получить доступ к DOM-узлу внутри вашего кастомного компонента из родителя, необходимо использовать специальный механизм (например, прокидывание пропса ref дочернему HTML-элементу)
