# Side Effects в React: Понятие, особенности и примеры

Side Effects (побочные эффекты) — это любые действия, которые выходят за рамки простого вычисления интерфейса и влияют на внешние системы или состояние вне компонента. В React функции компонентов должны быть «чистыми»: получать входные данные (props) и возвращать разметку (JSX), не меняя ничего снаружи. Если компоненту нужно сделать что-то еще, это считается побочным эффектом

### Что относится к Side Effects?

Типичными примерами побочных эффектов являются:

- **Сетевые запросы**: получение данных с сервера через API
- **Ручное манипулирование DOM**: использование браузерных методов, таких как .focus() или вызов сторонних библиотек (например, Google Maps или jQuery-виджетов)
- **Таймеры и интервалы**: установка setTimeout или setInterval
- **Подписки**: установка соединения с чат-сервером или подписка на события браузера
- **Логирование**: отправка аналитических данных при отображении компонента

### Основной инструмент: хук useEffect

Для управления побочными эффектами, вызванными самим рендерингом, используется хук useEffect. Он позволяет «отложить» выполнение кода до того момента, когда изменения будут зафиксированы в DOM и отображены на экране

#### Ключевые составляющие useEffect:

- **Код эффекта**: сама логика, которая выполняется после рендера
- **Массив зависимостей**: список значений, за которыми следит React. Эффект перезапустится только в том случае, если хотя бы одно из этих значений изменилось
  - Если массив пустой ([]), эффект выполнится только один раз при «монтировании» компонента
  - Если массива нет, эффект будет запускаться после каждого рендера
- **Функция очистки (Cleanup)**: функция, которую эффект может вернуть. React вызывает её перед повторным запуском эффекта или при удалении компонента с экрана, чтобы избежать утечек памяти или лишних соединений

### Эффекты vs Обработчики событий

Важно различать, когда использовать эффект, а когда — обычный обработчик (например, onClick или onSubmit):

- **События вызываются действием пользователя**: Если код должен сработать, потому что пользователь нажал на кнопку (например, «Купить товар»), его нужно поместить в обработчик события
- **Эффекты вызываются рендерингом**: Если код должен сработать просто потому, что компонент появился на экране (например, установка соединения с чатом), это работа для useEffect

### Философия «Вы можете обойтись без эффекта»

Одна из важнейших ментальных моделей уверенного разработчика — минимизация использования useEffect. Случаи, когда эффекты избыточны:

- **Трансформация данных**: если вы можете вычислить значение на основе существующих пропсов или состояния (например, полное имя из имени и фамилии), делайте это прямо в теле функции компонента во время рендеринга
- **Синхронизация состояний**: использование эффекта для изменения одного состояния при изменении другого часто ведет к лишним перерисовкам и запутанному коду

Понимание того, как и когда синхронизировать компоненты с внешними системами, является критическим навыком для перехода на Middle-уровень

### Примеры использования useEffect

#### 1. Загрузка данных с API

Этот пример показывает, как использовать useEffect для получения данных при монтировании компонента

```tsx
import { useState, useEffect } from "react";

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Эффект запускается при изменении userId
    setLoading(true);

    fetch(`https://api.example.com/users/${userId}`)
      .then((response) => response.json())
      .then((data) => {
        setUser(data);
        setLoading(false);
      });
  }, [userId]); // Эффект перезапустится при изменении userId

  if (loading) return <p>Загрузка...</p>;
  return <div>Имя пользователя: {user.name}</div>;
}
```

#### 2. Установка таймера с очисткой

Пример показывает, как правильно очищать ресурсы, чтобы избежать утечек памяти

```tsx
import { useState, useEffect } from "react";

function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    // Устанавливаем интервал
    const intervalId = setInterval(() => {
      setSeconds((s) => s + 1);
    }, 1000);

    // Функция очистки: вызывается при размонтировании
    return () => {
      clearInterval(intervalId);
    };
  }, []); // Пустой массив = эффект запустится один раз

  return <div>Прошло секунд: {seconds}</div>;
}
```

#### 3. Подписка на события браузера

Эффект для отслеживания размеров окна с правильной очисткой подписки

```tsx
import { useState, useEffect } from "react";

function WindowSize() {
  const [width, setWidth] = useState(window.innerWidth);

  useEffect(() => {
    // Обработчик изменения размера окна
    const handleResize = () => {
      setWidth(window.innerWidth);
    };

    // Подписываемся на событие
    window.addEventListener("resize", handleResize);

    // Очистка: удаляем обработчик при размонтировании
    return () => {
      window.removeEventListener("resize", handleResize);
    };
  }, []); // Эффект запускается один раз

  return <p>Ширина окна: {width}px</p>;
}
```

#### 4. Фокус на поле ввода при монтировании

Пример прямого манипулирования DOM через useEffect

```tsx
import { useEffect, useRef } from "react";

function SearchInput() {
  const inputRef = useRef(null);

  useEffect(() => {
    // Устанавливаем фокус после появления компонента на экране
    inputRef.current?.focus();
  }, []); // Выполнится один раз при монтировании

  return <input ref={inputRef} type="text" placeholder="Начните вводить..." />;
}
```

#### 5. Синхронизация с внешней системой (чат)

Пример подключения к внешнему сервису с правильной очисткой

```tsx
import { useEffect } from "react";

function ChatRoom({ roomId }) {
  useEffect(() => {
    // Подключение к чату
    const connection = createConnection(roomId);
    connection.connect();

    // Очистка: отключение при размонтировании или смене roomId
    return () => {
      connection.disconnect();
    };
  }, [roomId]); // Переподключаемся при смене комнаты

  return <div>Комната чата: {roomId}</div>;
}
```

### Когда использовать useEffect

Используйте useEffect для:

- Синхронизации компонента с внешними системами (API, браузерные API, сторонние библиотеки)
- Выполнения действий после того, как компонент отобразился на экране
- Подписки на события и установки соединений с правильной очисткой

**Избегайте useEffect для:**

- Трансформации данных для рендеринга (делайте это прямо в теле компонента)
- Обработки пользовательских событий (используйте обработчики событий)
- Синхронизации состояний внутри компонента (используйте производные значения или useReducer)
