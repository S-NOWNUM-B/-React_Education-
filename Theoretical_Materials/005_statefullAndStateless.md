# Statefull and Stateless: что это такое, в чём разница и где применять

В архитектуре React компоненты разделяются на два основных типа в зависимости от того, как они управляют данными: Stateful (с состоянием) и Stateless (без состояния). Это разделение помогает создавать более предсказуемый и легкий в поддержке код

---

### Stateless компоненты (Компоненты без состояния)

Такие компоненты также называют «презентационными» или «глупыми». Их главная задача — просто отрисовать UI на основе полученных данных

- Как работают: Они получают информацию исключительно через пропсы (props) от родительских компонентов
- Особенности: Они не знают, откуда пришли данные и как они меняются. У них нет своего «мнения» или «памяти»
- Преимущества: Их очень легко тестировать и переиспользовать в разных частях приложения, так как результат их работы полностью зависит от входных параметров

### Пример Stateless:

```jsx
function welcomeLabel({ name }) {
  return <h1>Привет, {name}!</h1>;
}
```

---

### Stateful компоненты (Компоненты с состоянием)

Эти компоненты называют «умными» или компонентами-контейнерами. Они отвечают за логику и хранение данных, которые могут меняться со временем

- Как работают: Они используют хуки, такие как useState или useReducer, чтобы «запоминать» информацию между рендерами
- Особенности: Они реагируют на действия пользователя (клик, ввод текста), обновляют свое состояние и передают его вниз дочерним stateless-компонентам
- Назначение: Служат «источником истины» для части интерфейса

### Пример Statefull:

```jsx
import { useState } from "react";

function NameForm() {
  // Компонент "помнит" введенное имя
  const [name, setName] = useState("");

  return <input value={name} onChange={(e) => setName(e.target.value)} />;
}
```

---

### Управляемые и неуправляемые компоненты
Понятия Stateful и Stateless тесно связаны с концепцией контроля:
1) Управляемый компонент (Controlled): Ведет себя как Stateless. Он не хранит состояние сам, а получает текущее значение и функцию изменения через пропсы от родителя. Родитель полностью диктует ему, что отображать
2) Неуправляемый компонент (Uncontrolled): Ведет себя как Stateful. Он сам хранит свои данные (например, через локальный useState), и родитель не может напрямую влиять на его внутренности без изменения пропсов-идентификаторов (например, key)

---

### Почему это важно?

В профессиональной разработке, которой обучают на таких платформах, как Stepik, рекомендуется поднимать состояние (lifting state up) к ближайшему общему родителю. Это позволяет превращать множество сложных Stateful-компонентов в простые Stateless-компоненты, делая поток данных в приложении прозрачным и односторонним
