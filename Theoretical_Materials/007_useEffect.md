# useEffect в React: Синхронизация с внешними системами

useEffect — это встроенный React-хук, который позволяет функциональным компонентам выполнять побочные эффекты (side effects). Он предназначен для синхронизации вашего компонента с системами вне React, такими как сетевые запросы, браузерный DOM или сторонние библиотеки

### Основное назначение

В отличие от обработчиков событий (например, onSubmit), которые запускаются в ответ на конкретное действие пользователя, Эффекты запускаются автоматически после того, как компонент был отрисован на экране. Это «инструмент побега» (escape hatch) из чистого мира React, позволяющий выйти за рамки простого вычисления интерфейса

### Как работает useEffect: три шага

Для правильного написания Эффекта необходимо выполнить следующие действия:

- **Объявить Эффект**: поместить логику внутрь useEffect. Код выполнится после того, как изменения будут зафиксированы в DOM
- **Указать зависимости**: определить, в каких случаях Эффект должен перезапускаться
- **Добавить очистку** (при необходимости): вернуть функцию, которая «отменит» действия Эффекта (например, закроет соединение)

### Управление частотой запусков (Массив зависимостей)

Поведение хука напрямую зависит от второго аргумента — массива зависимостей:

- **Без массива**: Эффект запускается после каждого рендера. Это может привести к проблемам с производительностью или бесконечным циклам
- **Пустой массив []**: Эффект запускается только один раз, когда компонент впервые появляется на экране (монтируется)
- **С переменными [prop, state]**: Эффект запускается при монтировании и каждый раз, когда изменяется хотя бы одно из указанных значений

### Функция очистки (Cleanup function)

Многие Эффекты требуют удаления за собой, чтобы избежать утечек памяти. Если ваш Эффект делает что-то асинхронное или создает подписку, он должен возвращать функцию очистки

- React вызывает функцию очистки перед каждым новым запуском Эффекта и один последний раз при удалении компонента (размонтировании)
- В режиме разработки React специально запускает и очищает Эффект дважды, чтобы помочь вам найти ошибки в логике очистки

### Когда Эффекты НЕ нужны (You Might Not Need an Effect)

Одной из характеристик опытного разработчика (уровня Confident Developer) является умение минимизировать использование useEffect. Выделяют два случая, когда Эффект избыточен:

- **Трансформация данных**: если значение можно вычислить на основе существующих пропсов или состояния (например, полное имя из имени и фамилии), это нужно делать прямо во время рендеринга, а не в Эффекте
- **Обработка событий пользователя**: логика, которая должна сработать в ответ на клик или отправку формы, должна находиться в соответствующем обработчике события, а не в Эффекте

### Примеры использования useEffect

#### 1. Загрузка данных с API

Этот пример показывает, как использовать useEffect для получения данных при появлении компонента на экране

```tsx
import { useState, useEffect } from "react";

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    setLoading(true);

    fetch(`https://api.example.com/users/${userId}`)
      .then((response) => response.json())
      .then((data) => {
        setUser(data);
        setLoading(false);
      });
  }, [userId]);

  if (loading) return <div>Загрузка...</div>;
  return <div>{user?.name}</div>;
}
```

#### 2. Подписка на событие браузера

Пример управления слушателями событий с обязательной функцией очистки

```tsx
import { useState, useEffect } from "react";

function WindowSize() {
  const [width, setWidth] = useState(window.innerWidth);

  useEffect(() => {
    const handleResize = () => setWidth(window.innerWidth);

    window.addEventListener("resize", handleResize);

    return () => {
      window.removeEventListener("resize", handleResize);
    };
  }, []);

  return <div>Ширина окна: {width}px</div>;
}
```

#### 3. Управление DOM элементами

Синхронизация состояния компонента с атрибутами документа (например, title страницы)

```tsx
import { useState, useEffect } from "react";

function PageTitle({ title }) {
  useEffect(() => {
    document.title = title;
  }, [title]);

  return <h1>{title}</h1>;
}
```

#### 4. Таймер с правильной очисткой

Пример использования setInterval с функцией очистки для предотвращения утечек памяти

```tsx
import { useState, useEffect } from "react";

function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const interval = setInterval(() => {
      setSeconds((prev) => prev + 1);
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return <div>Прошло секунд: {seconds}</div>;
}
```

Понимание жизненного цикла Эффектов и правил их использования является обязательным навыком для работы в крупных проектах
