# useState в React: Понятие, особенности и примеры

useState — это специальный хук React, который позволяет функциональным компонентам «запоминать» информацию, например, ввод пользователя или состояние интерфейса. В то время как обычные переменные сбрасываются при каждом цикле отрисовки, состояние (state) сохраняется между повторными рендерами компонента

### Что такое useState и как он работает

При вызове useState вы сообщаете React, что компонент должен хранить определенное значение. Хук всегда возвращает массив из двух элементов: текущее значение состояния и функцию-сеттер для его обновления

### Ключевые особенности useState

- Инициирование рендеринга: Обновление состояния через функцию-сеттер (например, setIndex) автоматически ставит в очередь новый процесс отрисовки компонента, чтобы отобразить актуальные данные
- Изоляция и приватность: Состояние полностью локально для каждого экземпляра компонента. Если вы отрисуете один и тот же компонент в двух местах, каждый из них будет иметь свою независимую «память»
- Правила вызова: Хуки можно вызывать только на верхнем уровне компонента. Их нельзя помещать внутрь условий, циклов или вложенных функций, так как React полагается на стабильный порядок их вызова
- Батчинг (Группировка): React не обновляет экран мгновенно после каждого вызова сеттера. Вместо этого он ждет завершения всего кода в обработчике событий и выполняет обновления одной группой для повышения производительности
- Функции обновления: Если вам нужно обновить состояние несколько раз подряд в одном событии, следует передавать функцию обновления (updater function), например: setNumber(n => n + 1)

### Примеры использования useState

#### 1. Простой счетчик (числовое состояние)

Этот пример показывает, как useState удерживает значение при кликах и заставляет React перерисовать интерфейс

```tsx
import { useState } from "react";

function Counter() {
  // 0 — начальное значение. count — текущее, setCount — функция для изменения
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>Вы нажали {count} раз</button>
  );
}
```

#### 2. Переключатель (логическое состояние)

Состояние часто используется для хранения флагов «включено/выключено»

```tsx
function Toggle() {
  const [isOn, setIsOn] = useState(false);

  return (
    <button onClick={() => setIsOn(!isOn)}>
      {isOn ? "Включено" : "Выключено"}
    </button>
  );
}
```

#### 3. Работа с полем ввода (строковое состояние)

Состояние позволяет компоненту «помнить», что именно ввел пользователь

```tsx
function SearchForm() {
  const [query, setQuery] = useState("");

  return (
    <input
      value={query}
      onChange={(e) => setQuery(e.target.value)}
      placeholder="Поиск..."
    />
  );
}
```

### Когда использовать useState

Используйте useState, если информация должна влиять на то, что отображается на экране. Если же данные нужны только для технических целей (например, хранить ID таймера) и не требуют перерисовки интерфейса, лучше использовать useRef. Для более сложной логики обновлений, затрагивающей множество связанных данных, React предоставляет альтернативный хук useReducer
