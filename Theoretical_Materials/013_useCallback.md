# useCallback в React: Кэширование функций

useCallback — это встроенный React-хук, который позволяет сохранять одну и ту же ссылку на функцию между повторными рендерами компонента. Это помогает избежать лишнего создания новых функций и используется в задачах оптимизации производительности.

### Как работает useCallback

Хук принимает два аргумента: функцию и массив зависимостей.

- **На первом рендере**: React возвращает ту функцию, которую вы передали
- **На следующих рендерах**: React сравнивает зависимости с предыдущими значениями
- **Если зависимости не изменились**: возвращается та же ссылка на функцию
- **Если зависимости изменились**: React создает и сохраняет новую версию функции

### Когда useCallback полезен

На практике хук чаще всего нужен в трех случаях:

- **Оптимизация компонентов с memo**: Если в мемоизированный дочерний компонент передается функция, useCallback помогает не менять ее ссылку без необходимости
- **Стабилизация зависимостей в useEffect**: Если эффект зависит от функции, useCallback помогает избежать лишних повторных запусков
- **Кастомные хуки**: Если собственный хук возвращает функции, useCallback делает их более удобными для дальнейшей оптимизации в компонентах-потребителях

### Примеры использования useCallback

#### 1. Передача функции в мемоизированный компонент

```tsx
import { memo, useCallback } from "react";

const ShippingForm = memo(function ShippingForm({ onSubmit }) {
  return <form onSubmit={onSubmit}>...</form>;
});

function ProductPage({ productId }) {
  const handleSubmit = useCallback(
    (details) => {
      post("/buy/" + productId, details);
    },
    [productId],
  );

  return <ShippingForm onSubmit={handleSubmit} />;
}
```

#### 2. Обновление состояния без лишних зависимостей

Чтобы уменьшить количество зависимостей, можно использовать updater-функцию состояния.

```tsx
import { useCallback, useState } from "react";

let nextId = 1;

function TodoList() {
  const [todos, setTodos] = useState([]);

  const handleAddTodo = useCallback((text) => {
    const newTodo = { id: nextId++, text };

    setTodos((prevTodos) => [...prevTodos, newTodo]);
  }, []);

  return (
    <button onClick={() => handleAddTodo("Новая задача")}>Добавить</button>
  );
}
```

### Отличие useCallback от useMemo

- **useCallback** кэширует саму функцию
- **useMemo** кэширует результат вычисления

Если вам нужно сохранить стабильную ссылку на функцию, используйте useCallback. Если нужно сохранить результат вычисления, используйте useMemo.

### Важные правила использования

- **Только на верхнем уровне**: Как и другие хуки, useCallback нельзя вызывать внутри условий, циклов или вложенных функций
- **Не использовать без причины**: Для простых компонентов создание новой функции обычно ничего не ломает и почти ничего не стоит
- **Ориентироваться на реальную пользу**: useCallback нужен там, где он помогает memo, useEffect или пользовательским хукам
- **React Compiler**: В новых версиях React часть таких оптимизаций может выполняться автоматически

### Заключение

useCallback нужен для сохранения стабильных ссылок на функции и особенно полезен при работе с memo, useEffect и кастомными хуками. Как и другие инструменты мемоизации, его стоит применять не повсеместно, а только там, где он действительно улучшает производительность или делает поведение кода стабильнее.
