# memo в React: Оптимизация производительности компонентов

memo — это API React, которое позволяет пропускать повторный рендеринг компонента, если его пропсы не изменились. Это один из основных инструментов оптимизации производительности в React-приложениях.

### Что такое memo и как он работает

По умолчанию React повторно вызывает дочерний компонент каждый раз, когда рендерится его родитель. Если обернуть компонент в memo, React будет сравнивать его новые пропсы с предыдущими и решать, нужен ли новый рендер.

- **Если пропсы не изменились**: React может пропустить повторный рендер компонента и использовать предыдущий результат
- **Если хотя бы один пропс изменился**: компонент будет отрисован заново

### Основные особенности memo

- **Поверхностное сравнение**: По умолчанию React сравнивает пропсы через Object.is. Если передать новый объект, массив или функцию, созданные прямо во время рендера, memo посчитает такой пропс изменившимся
- **Это оптимизация, а не гарантия**: memo помогает сократить количество рендеров, но не обещает, что React никогда не вызовет компонент повторно
- **Связь с useMemo и useCallback**: Чтобы memo работал эффективно, в родителе часто приходится стабилизировать объекты и функции через useMemo и useCallback
- **React Compiler**: В современных версиях React часть такой оптимизации может выполняться автоматически, поэтому ручная мемоизация нужна не всегда

### Когда memo действительно полезен

memo стоит использовать там, где повторные рендеры реально мешают производительности:

- В тяжелых компонентах с заметной стоимостью рендера
- В больших списках, где каждый элемент может рендериться часто
- В интерфейсах, где родитель обновляется часто, а часть дочерних компонентов фактически не меняется

### Примеры использования memo

#### 1. Базовое использование

Компонент Greeting не будет перерисовываться, если родитель меняет состояние, не связанное с пропсом name.

```tsx
import { memo, useState } from "react";

const Greeting = memo(function Greeting({ name }) {
  console.log("Рендер Greeting");
  return <h1>Привет, {name}!</h1>;
});

export default function App() {
  const [name, setName] = useState("");
  const [count, setCount] = useState(0);

  return (
    <>
      <input value={name} onChange={(e) => setName(e.target.value)} />
      <button onClick={() => setCount(count + 1)}>Счетчик: {count}</button>
      <Greeting name={name} />
    </>
  );
}
```

#### 2. Когда memo не поможет

Если передавать объект или функцию напрямую без стабилизации ссылки, memo будет бесполезен, потому что такой пропс считается новым на каждом рендере.

```tsx
import { memo, useState } from "react";

const Status = memo(function Status({ config }) {
  return <div style={{ color: config.color }}>Статус</div>;
});

function Parent() {
  const [value, setValue] = useState(0);

  return (
    <>
      <button onClick={() => setValue(value + 1)}>Обновить</button>
      <Status config={{ color: "red" }} />
    </>
  );
}
```

### Заключение

memo помогает уменьшить количество лишних рендеров, но применять его стоит осознанно. Это инструмент оптимизации, а не обязательная часть каждого компонента. Наибольшую пользу он приносит там, где есть реальные проблемы с производительностью и стабильность пропсов действительно имеет значение.
