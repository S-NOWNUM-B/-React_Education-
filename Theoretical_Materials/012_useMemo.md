# useMemo в React: Кэширование вычислений

useMemo — это встроенный React-хук, который позволяет кэшировать результат вычислений между повторными рендерами компонента. Он помогает избегать повторного выполнения дорогих операций, если зависимости не изменились.

### Как работает useMemo

Хук принимает два аргумента:

- **Функцию расчета**: Чистая функция без аргументов, которая возвращает вычисленное значение
- **Массив зависимостей**: Список значений, от которых зависит результат вычисления

Во время первого рендера React вызывает функцию расчета и запоминает результат. На следующих рендерах React сравнивает зависимости с предыдущими значениями через Object.is.

- **Если зависимости не изменились**: возвращается сохраненное значение
- **Если хотя бы одна зависимость изменилась**: вычисление выполняется заново

### Когда useMemo действительно нужен

Этот хук полезен не всегда, а в конкретных ситуациях:

- **Тяжелые вычисления**: Например, фильтрация, сортировка или преобразование больших массивов данных
- **Стабилизация пропсов для memo**: Если вы передаете объект или массив в компонент, обернутый в memo, useMemo помогает сохранить ту же ссылку между рендерами
- **Стабильность зависимостей для других хуков**: Мемоизированное значение может использоваться в useEffect или другом useMemo без лишних повторных срабатываний

### Пример использования useMemo

В этом примере фильтрация списка выполняется только тогда, когда действительно меняются items или category.

```tsx
import { useMemo } from "react";

function ProductList({ items, category }) {
  const filteredItems = useMemo(
    () => items.filter((item) => item.type === category),
    [items, category],
  );

  return (
    <ul>
      {filteredItems.map((item) => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}
```

### Важные особенности и ограничения

- **Не оптимизировать преждевременно**: Во многих случаях вычисления и так очень быстрые, а сам useMemo тоже имеет свою стоимость
- **Функция должна быть чистой**: Внутри useMemo нельзя выполнять побочные эффекты, такие как запросы к API, таймеры или прямую работу с DOM
- **Strict Mode**: В режиме разработки React может вызывать функцию расчета дважды, чтобы выявлять случайные побочные эффекты
- **React Compiler**: В новых версиях React часть мемоизации может выполняться автоматически, поэтому ручное использование useMemo требуется не всегда

### Отличие useMemo от useCallback

- **useMemo** кэширует результат вычисления
- **useCallback** кэширует саму функцию

Это различие важно: если вам нужно сохранить массив, объект или число после вычисления, нужен useMemo. Если требуется сохранить стабильную ссылку на функцию, нужен useCallback.

### Заключение

useMemo полезен для оптимизации вычислений и стабилизации значений между рендерами, но его стоит применять только там, где это действительно оправдано. Грамотное использование мемоизации помогает делать интерфейс быстрее и уменьшать лишние перерисовки.
